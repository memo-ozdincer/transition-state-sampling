\section{Summary}
%=============================================================================

Two-phase optimizer: Newton--Raphson (NR) minimization $\to$ Gentlest Ascent
Dynamics (GAD) saddle search.  Given a noisy geometry near a transition state,
Phase~1 descends to a local minimum, Phase~2 climbs back to an index-1 saddle.

\textbf{Convergence criterion:} $n_\text{neg} = 1$ (exactly one negative
eigenvalue on the Eckart-projected Hessian).  No threshold filtering.
See Appendix~\ref{app:criterion} for details and bug-fix history.

\medskip
\noindent\colorbox{yellow!30}{\parbox{0.93\textwidth}{%
\textbf{Data status (2026-03-24):} All numerical results below were collected
with a \emph{wrong} convergence criterion ($\eval_0 \times \eval_1 < -\epsilon$
with \texttt{tr\_threshold} filtering).  The code is now fixed to $n_\text{neg}=1$.
Treat TS rates as \textbf{upper bounds}; re-runs are in progress.
Qualitative findings (nopath $\gg$ path) are expected to hold.
}}

%=============================================================================
\section{Method}
\label{sec:method}
%=============================================================================

\begin{description}[nosep,leftmargin=1.5cm,style=sameline]
  \item[Phase 1] NR minimization (RFO + polynomial line search).
        Converges when Morse index~$= 0$ and forces $< 10^{-4}$~eV/$\angstrom$.
  \item[Phase 2] GAD saddle search from the converged minimum.
        Converges when $n_\text{neg} = 1$ on the Eckart-projected Hessian.
\end{description}

\noindent Two GAD variants:
\begin{itemize}[nosep]
  \item \textbf{Path}: trust-region step control, full feature set.
  \item \textbf{Nopath}: state-based, eigenvalue-clamped $dt$ adaptation,
        Markovian (no trajectory history)---suitable for generative modelling.
\end{itemize}

Dataset: DFTB0 on Transition1x ($N \approx 287$ samples, 8 molecular families).
Noise levels $\sigma \in \{0.5, 1.0, 1.5, 2.0\}$~$\angstrom$ applied to
midpoint geometries.  Budgets: 50k NR steps, 20k GAD steps.

\subsection{Why path GAD fails: trust-region overshoot}

Path GAD failures land at Morse index 2--8, not index~1.
The trust-region step size overshoots past the index-1 saddle.
At $\sigma = 2.0$, 75\% of path failures are \texttt{singularity\_high\_index}
(overshoot to higher-index critical points).

\begin{figure}[H]

\subsection{Why nopath GAD works: self-regulating step size}

The eigenvalue-clamped step size $dt_\text{eff} = dt \cdot s / \text{clamp}(|\eval_0|)$
naturally shrinks when $|\eval_0|$ is large (preventing overshoot) and grows
when $|\eval_0| \to 0$ (enabling escape from minima).  This creates a damped
oscillation that converges to the TS.

\section{Conclusions}
\label{sec:conclusions}
%=============================================================================

\noindent\textit{Provisional---based on old-criterion data.  Qualitative
ranking confirmed by 3-sample smoke test with the corrected
$n_\text{neg}=1$ criterion (nopath converges to true Morse-1 saddles,
mean $\sim$566 GAD steps).}

\begin{enumerate}[nosep]
  \item \textbf{Nopath GAD dominates path GAD} on TS rate ($\geq$95\% vs.\
        75\%$\to$15\%), wall-clock time (2--3$\times$ faster), and pipeline
        simplicity (Markovian, no trajectory history).

  \item \textbf{The bottleneck is NR, not GAD.}
        End-to-end TS rate $\approx$ NR convergence rate $\times$ 95--100\%.

  \item \textbf{GAD is cheap.}
        Median 9--34 steps; $<$3\% of total pipeline budget.

  \item \textbf{Universal across molecular families.}
        All 8 formulas achieve $\geq$98.7\% TS rate (given NR convergence).

  \item \textbf{Path GAD's trust-region is fundamentally flawed for this task.}
        Overshoots to Morse 2--8; rescue mode provides zero benefit.
\end{enumerate}

\subsection{Recommended pipeline for generative modelling}

\begin{enumerate}[nosep]
  \item Simple NR (RFO + PLS, no kicks), 50k steps.
  \item Nopath GAD (eigenvalue-clamped $dt$), 2\,000 steps.
  \item Expected yield: 93--95\% end-to-end TS rate.
\end{enumerate}

Every step is a pure function of $(\text{coords}, E, \vect{F}, \hess)$---no
trajectory history, suitable for learning the optimization policy.

\section{Convergence Criterion: Details and Bug Fix}
\label{app:criterion}

\subsection{Current criterion}

The GAD phase declares ``TS found'' when
\begin{equation}
  n_\text{neg} = 1 \qquad \text{(Morse index 1)},
\end{equation}
where $n_\text{neg}$ counts negative eigenvalues of the Eckart-projected
Hessian.  No additional \texttt{tr\_threshold} filtering is applied.

\subsection{Eigenvalue product cascade (diagnostic only)}

At convergence, the eigenvalue product $\eval_0 \times \eval_1$ is logged
for three thresholds ($10^{-3}$, $10^{-5}$, $10^{-8}$).  These show how
many samples would converge under weaker definitions of ``TS'' but are
\emph{not} used for convergence.

\subsection{Bug fixed (2026-03-24)}

The original code used $\eval_0 \times \eval_1 < -\epsilon_\text{ts}$
($\epsilon_\text{ts} = 10^{-5}$) as the convergence gate, with
\texttt{tr\_threshold} = $8 \times 10^{-3}$ filtering eigenvalues first.
This was too lenient---it accepted geometries with multiple small negative
eigenvalues as ``TS.''  Additionally, the logger used a different threshold
($10^{-6}$), causing Morse indices in trajectories to disagree with the
runner's convergence check.

\textbf{Fix applied to 7 files:}
\texttt{run\_gad\_baselines\_parallel\_nopath.py},
\texttt{run\_gad\_baselines\_parallel.py},
\texttt{gad\_fixed\_dt.py},
\texttt{multi\_mode\_eckartmw.py},
\texttt{run\_kick\_comparison.py},
\texttt{metrics\_nopath.py},
\texttt{metrics.py}.

\textbf{Smoke test (3 samples, new criterion):} Converged samples correctly
show Morse index~1.  Mean GAD steps increased from $\sim$12 to $\sim$566
(expected---the criterion is stricter).
